梯度下降

（开始写的时候已经看完了凭印象写的，有空再补充补充）

简单来说，就是瞎蒙了一个模型，求出模型的预测值和实际值的差距，即损失之后，我们肯定是想要让损失函数的值越小越好，所以对损失函数求导，然后朝着负梯度方向走步长（学习率）的距离，来让损失函数的取值越来越小（因为负梯度方向是下降最快的方向）。

之前用的那个SGD*，就是随机梯度下降，这是为了避免常规梯度下降的时候遇到了鞍点，或者一个大平面/一条大直线，那梯度肯定是0，梯度下降的时候，权重 - 学习率 * 梯度就会一直不变，加上随机之后就有概率冲出那片平坦的函数区域了。

关于batch_size，这涉及到损失函数的计算，以方差为例吧，普通的梯度下降，是选取所有测试点计算预测值与真实值的方差之和 除以 测试点的数量（懒得打公式了），这种情况下所有自变量在本次计算时使用的都是同一个时刻的权重，但是随机梯度下降，用的不是所有测试点，最普通的SGD是随机选取一个测试点计算权重，但是这样是没法并行计算的。所以我们折个中，把测试点分成一批批来算，即Mini Batch。

反向传播

  • forword：把X和w（权重）往下传，最终计算出预测值后可以算出loss

  • backword：把loss的导数往回传，再根据链导法则求出每一层的X和w的导数接着往回传

  • 最终目的是求得损失对权重的导数，这样就知道该如何更新权重了

（为什么X导数也要回传呢，因为尽管X是自变量，但是中间层的X导数是要传回去的，因为它是上一层的输出）

softmax:

在机器学习尤其是深度学习中，softmax是个非常常用而且比较重要的函数，尤其在多分类的场景中使用广泛。他把一些输入映射为0-1之间的实数，并且归一化保证和为1，因此多分类的概率之和也刚好为1。
首先我们简单来看看softmax是什么意思。顾名思义，softmax由两个单词组成，其中一个是max。对于max我们都很熟悉，比如有两个变量a,b。如果a>b，则max为a，反之为b。用伪码简单描述一下就是 if a > b return a; else b。
另外一个单词为soft。max存在的一个问题是什么呢？如果将max看成一个分类问题，就是非黑即白，最后的输出是一个确定的变量。更多的时候，我们希望输出的是取到某个分类的概率，或者说，我们希望分值大的那一项被经常取到，而分值较小的那一项也有一定的概率偶尔被取到，所以我们就应用到了soft的概念，即最后的输出是每个分类被取到的概率。

预测/NMS后处理

每一个grid cell包含了：

  • 两个由当前cell生成的预测框的信息（x, y, w, h, c/置信度）

  • 20个对于每一种类别的概率信息（一共20种类别）

  • （每个cell最后最多只能预测出一个物体）

当前bounding box包含x物体的概率，就是该bbox的置信度再乘上当前bbox对应cell对于x类别的概率；每个bbox可以生成20个这样的概率，每个cell对应两个bbox，因此最后会得到一个7 * 7 * 2 * 20的向量：

之后，需要对数据进行过滤筛选。考虑这每一个box对应的向量里是狗的概率的那个分量，过程如下：

  1. 设定一个阈值，把低于阈值概率的一律设为0

  2. 按照是狗的概率的高低来排序

  3. 对上一步得到的向量组进行NMS（非极大值抑制）

NMS过程：

  1. 取出最大的概率对应的bbox

  2. 之后的bbox依次与最大bbox计算IOU（交并比），如果IOU高于某个阈值，则认为是重叠的，去掉当前的重复bbox（概率设为0）

  3. 第一高的和其余的依次比对完后，再从第二高的开始（注意在上例中，bb20已经不是第二高的了，而是永远地消失了）

  4. 最后，会剩下一个或者若干个框，它们的预测准确的概率相对较高并且不是很重合

训练

训练的时候有人工标注的框框，和框框的中心点。训练的时候要让那个位置的bbox尽量去拟合人工标注的框框（ground truth）。上文中说到了一个grid cell预测出两个bbox，我们要采用那个和ground truth的IOU更大的那个来作为结果，如果某个grid cell预测的两个bbox都没有包含ground truth的中心点，那么这两个bbox都应该被弃用（p.s. grid cell预测出的两个bbox不管多大，它们的中心点肯定是落在grid cell之内的）

一些Score

查准率和查全率是在信息检索、Web搜索等应用中经常出现的性能度量指标，在机器学习中，也可以用来度量“预测结果中有多少比例是用户感兴趣的”。对于二分类问题，可将样例根据其真实类别与预测类别的组合划分为：TP、TN、FP、FN

  • TP (True Positive)：真正例。Positive指预测输出为正，True代表预测正确。

  • TN (True Negative)：真反例。预测输出为负，而且预测正确。

  • FP (False Positive)：假正例。预测输出为正，但是预测错误。

  • FN (False Negative)：假反例。预测输出为负，但是预测错误。

  • Precision(查准率)：所有预测为正例的结果中，预测正确的比率。

  • Recall (查全率)：所有正例中被正确预测的比率。

Precision score表示模型对所有正预测中正确预测正数的能力。Precision score是衡量类平衡时预测成功的有用指标。在数学上，它表示真阳性与真阳性和假阳性之和的比率：Precision score = TP / (FP + TP)

Recall Score（召回率分数）表示模型从实际阳性样本中正确预测阳性的能力。这与精确度不同，精确度衡量模型做出的预测在所有正预测中有多少实际上是正的。例如：如果你的机器学习模型试图识别正面评价，那么Recall Score将是你的机器学习模型正确预测为正面评价的正面评价的百分比。换句话说，它衡量我们的机器学习模型在从数据集中存在的所有积极因素中识别出所有实际积极因素方面有多好。召回率得分越高，机器学习模型在识别正面和负面样本能力就越好。Recall Score是衡量类别非常不平衡时预测正确的有用指标。 在数学上，它表示真阳性与真阳性和假阴性之和的比率：Recall Score = TP / (FN + TP)

Accuracy Score定义为真阳性和真阴性与所有正值和负值的比率。换句话说，准确性告诉我们，我们可以期望我们的机器学习模型从它做出预测的总次数中正确预测结果的频率。例如：假设你正在使用包含100条记录的数据集测试机器学习模型，并且机器学习模型正确预测了所有90个实例。在本例中，准确性指标为：（90/100） = 90%。准确率很高，但它并没有告诉我们机器学习模型对我们以前从未见过的新数据会产生较大的错误。在数学上，它表示所有预测中真正和真负之和占所有样本的比率：Accuracy Score = (TP + TN) / (TP + FN + TN + FP)

目标检测问题中，算法通常输出一个预测框(bounding box)来标识检出目标的位置，要衡量这个预测框与目标的实际位置(ground truth)的准确度，可以使用IoU指标。

IoU（交并比，Intersection over union）衡量的是两个区域的重叠程度，是两个区域重叠部分面积占二者总面积(重叠部分只计算一次)的比例。目标检测中IoU就是预测框与实际框的交集除以并集。

我们可以设置一个阈值(threshold)，通常是0.5，预测结果可以分为：

  • 如果IoU >= 0.5

    • 如果预测类别也正确，认为是一个好的预测，分类为TP

    • 如果预测类别错误，认为是一个坏的预测，分类为FP

  • 如果IoU < 0.5, 认为是一个坏的预测，分类为FP

  • 如果一个目标出现在图像中，但是算法未检出，分类为FN

TN（图像上所有不包含实际框和检测框的部分）通常在计算中用不到。

查全率和查准率通常是一对矛盾的度量，一般来说，查准率高时，查全率往往偏低；而查全率高时，查准率往往偏低。

我们如果把所有预测结果对样例进行排序，排在前面的是“最可能”的正例样本，以查准率为纵轴、查全率为横轴作图，得到一条”P-R曲线”。

P-R曲线直观的显示出一个算法在样本总体上的查全率、查准率情况。如果一个算法的P-R曲线被另一个算法的曲线完全包住，则可断言后者优于前者。但是实际中，经常不同算法的P-R曲线是互相交叉的，这时就很难直观判断出两者的优劣。这时通常会考察平衡点(BEP)、F1度量、AP等指标。

AP（平均精度，Average Precision）：AP是计算单类别的模型平均准确度。对于目标检测任务，每一个类都可以计算出其Precision和Recall，每个类都可以得到一条P-R曲线，曲线下的面积就是AP的值。如果一个算法的AP值较大，也就是P-R曲线下的面积比较大，可以认为此算法查准率和查全率整体上相对

mAP（mean of Average Precision）：对所有类别的AP值求平均值。
