梯度下降

（开始写的时候已经看完了凭印象写的，有空再补充补充）

简单来说，就是瞎蒙了一个模型，求出模型的预测值和实际值的差距，即损失之后，我们肯定是想要让损失函数的值越小越好，所以对损失函数求导，然后朝着负梯度方向走步长（学习率）的距离，来让损失函数的取值越来越小（因为负梯度方向是下降最快的方向）。

之前用的那个SGD，就是随机梯度下降，这是为了避免常规梯度下降的时候遇到了鞍点，或者一个大平面/一条大直线，那梯度肯定是0，梯度下降的时候，权重 - 学习率 * 梯度就会一直不变，加上随机之后就有概率冲出那片平坦的函数区域了。

关于batch_size，这涉及到损失函数的计算，以方差为例吧，普通的梯度下降，是选取所有测试点计算预测值与真实值的方差之和 除以 测试点的数量（懒得打公式了），这种情况下所有自变量在本次计算时使用的都是同一个时刻的权重，但是随机梯度下降，用的不是所有测试点，最普通的SGD是随机选取一个测试点计算权重，但是这样是没法并行计算的。所以我们折个中，把测试点分成一批批来算，即Mini Batch。

神经网络

比如最简单的y = w * x + b，这就可以看作一个一层的神经网络，权重是w，偏置是b。当然，这个是1 x 1的矩阵，太简单了。如果我们输入的X是一个多维的向量，连接多层这样的神经网络后，根据矩阵乘法其实可以把这多层H1 = W1 * X + b1, H2 = W2 * H1 + b2, ..., Y = Wn * Hn + bn合并成一个式子Y = W * X + b，这显然属于分层分了个寂寞。再以2层的为例：

所以要在每一层对输出值再套一个非线性的激活函数，这样最后每一层就没法合并成一个总的线性函数了，借用弹幕的两句话：

激活函数给神经元引入了非线性因素，神经网络就可以逼近任意的非线性函数

不加的话，无论多少层都没有意义，因为到头来还是线性的

反向传播

  • forword：把X和w（权重）往下传，最终计算出预测值后可以算出loss

  • backword：把loss的导数往回传，再根据链导法则求出每一层的X和w的导数接着往回传

  • 最终目的是求得损失对权重的导数，这样就知道该如何更新权重了

（为什么X导数也要回传呢，因为尽管X是自变量，但是中间层的X导数是要传回去的，因为它是上一层的输出）

softmax:

在机器学习尤其是深度学习中，softmax是个非常常用而且比较重要的函数，尤其在多分类的场景中使用广泛。他把一些输入映射为0-1之间的实数，并且归一化保证和为1，因此多分类的概率之和也刚好为1。
首先我们简单来看看softmax是什么意思。顾名思义，softmax由两个单词组成，其中一个是max。对于max我们都很熟悉，比如有两个变量a,b。如果a>b，则max为a，反之为b。用伪码简单描述一下就是 if a > b return a; else b。
另外一个单词为soft。max存在的一个问题是什么呢？如果将max看成一个分类问题，就是非黑即白，最后的输出是一个确定的变量。更多的时候，我们希望输出的是取到某个分类的概率，或者说，我们希望分值大的那一项被经常取到，而分值较小的那一项也有一定的概率偶尔被取到，所以我们就应用到了soft的概念，即最后的输出是每个分类被取到的概率。
